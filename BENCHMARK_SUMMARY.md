## 一、业界主流框架调研

### 1.1 框架对比总览

| 框架 | 核心架构 | 特点 | 适用场景 |
|------|----------|------|----------|
| **OpenAI Deep Research** | 端到端强化学习 | Plan-Act-Observe 循环，自主回溯 | 通用深度研究 |
| **Perplexity Deep Research** | 三层 RAG 架构 | 混合检索 + 多阶段重排序 | 实时智能搜索 |
| **Microsoft Semantic Kernel** | 分层多 Agent | Manager + Specialists 协作 | 企业级应用 |
| **Jina AI node-DeepResearch** | Search-Read-Reason 循环 | 开源，高度可定制 | 开发者定制 |
| **LangGraph Research Agent** | 状态图工作流 | 自纠正，可观测性 | 可视化编排 |

### 1.2 核心架构模式

**业界标准架构**:
```
用户输入
  ↓
[查询理解] → 语义解析、实体抽取、意图分类
  ↓
[任务分解] → 子问题生成、研究计划
  ↓
[并行执行] → 多 Agent 协作、多源检索
  ↓
[观察评估] → 质量评估、策略调整 (ReAct)
  ↓
[事实核查] → 多源验证、冲突解决
  ↓
[中间摘要] → 子主题摘要、信息压缩
  ↓
[综合合成] → 报告生成、引用绑定
  ↓
[质量检查] → RACE/FACT 评估、格式验证
  ↓
用户输出
```

**当前实现架构**:
```
用户输入
  ↓
[实体解析] → 正则匹配、关键词提取
  ↓
[研究计划] → 问题分解、搜索任务
  ↓
[顺序执行] → 单 Agent、多轮检索
  ↓
[证据评估] → 多维度评分 (新增)
  ↓
[冲突检测] → 规则匹配 (新增)
  ↓
[综合合成] → 报告生成、引用列表
  ↓
用户输出
```

## 二、评估基准体系

### 2.1 主流评估框架

| 基准 | 评估维度 | 评分方式 | 行业领先水平 |
|------|----------|----------|--------------|
| **DeepResearch Bench** | RACE + FACT | 动态权重 | 8.51/10 |
| **DeepSearchQA** | 完整性、准确性 | 900 题基准 | - |
| **DeepResearchEval** | Coverage/Insight/Instruction/Citation | LLM-as-a-Judge | 8.51/10 |
| **HERO** | Key Point Recall, Citation F1 | 标准化指标 | 67.63 / 91.57 |

### 2.2 核心评估维度

1. **Coverage (覆盖率)**: 信息广度、深度、相关性
2. **Insight (洞察力)**: 分析深度、新颖见解
3. **Instruction Following (指令遵循)**: 格式、要求满足
4. **Citation Accuracy (引用准确性)**: 引用支持度
5. **Factual Reliability (事实可靠性)**: 事实核查正确率

### 5.2 具体行动项

#### P0 - 立即实施 (1-2 周)

   - 在最终合成前添加事实核查步骤
   - 标记未验证声明
   - 预期效果：错误信息传播 < 5%

3. **添加自动化评估到输出流程**
   - 在报告生成后运行 RACE/FACT 评估

   - 预期效果：质量可度量

#### P1 - 短期实施 (2-4 周)

4. **实现多 Agent 协作架构**
   - 设计 Manager Agent
   - 实现 Search/Analysis/Synthesis Agents
   - 预期效果：研究深度提升 25%

5. **实现并行执行框架**
   - 使用 asyncio/ThreadPoolExecutor
   - 子问题并行处理
   - 预期效果：研究时间减少 40%

6. **升级查询理解模块**
   - LLM 驱动语义解析
   - 实体关系抽取
   - 预期效果：实体识别准确率 > 90%

#### P2 - 中期实施 (4-8 周)

7. **实现智能 URL 选择**
   - 三阶段检索（搜索 → 评分 → 访问）
   - 预期效果：访问效率提升 50%

8. **添加中间摘要机制**
   - 子主题摘要
   - 信息压缩
   - 预期效果：上下文使用效率提升 40%

9. **实现用户反馈循环**
   - 反馈收集
   - 策略调整
   - 预期效果：用户满意度 > 4.0/5.0
